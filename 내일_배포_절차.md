# 내일 배포 절차

오늘 작업한 커뮤니티 Q&A 기능을 배포하는 순서입니다.
**두 단계 — Supabase 먼저, Git 나중.**

---

## 1단계. Supabase 테이블 생성 (먼저 해야 함)

> 이걸 먼저 해야 커뮤니티 페이지가 에러 없이 작동합니다.

1. [https://supabase.com](https://supabase.com) 접속 → 내 프로젝트 클릭
2. 왼쪽 사이드바 **SQL Editor** 클릭
3. `+ New query` 버튼 클릭
4. `C:\planing\ddaiga-americano\supabase_community_posts.sql` 파일을 텍스트 에디터로 열어서 **전체 복사**
5. SQL Editor에 붙여넣기 → **Run** 버튼 클릭
6. 왼쪽 사이드바 **Table Editor** 클릭 → `community_posts` 테이블이 생겼으면 성공

---

## 2단계. 로컬에서 정상 작동 확인 (선택이지만 권장)

```bash
cd C:\planing\ddaiga-americano
npm run dev
```

브라우저에서 확인할 것:

- `http://localhost:3000` → 홈 Q&A 섹션에 질문 카드가 보이는지
- `http://localhost:3000/community` → 질문 폼 + 목록이 뜨는지
- 네비게이션 바에 **Q&A** · **블로그** 링크가 있는지
- 질문을 하나 직접 등록해보고 Supabase Table Editor에서 데이터가 들어오는지 확인

## 3단계. Git add / commit / push

로컬 확인이 끝났으면 터미널에서 순서대로 실행:

```bash
cd C:\planing\ddaiga-americano

git add .

git commit -m "feat: 커뮤니티 Q&A 기능 추가

- components/HomeCommunitySection.tsx 추가 (홈 동적 Q&A 섹션)
- components/SiteHeader.tsx 네비에 Q&A / 블로그 링크 추가
- app/page.tsx HomeCommunitySection으로 교체
- app/shoe-finder/page.tsx FeedbackForm 제거"

git push
```

## 4단계. 배포 확인

Vercel 기준:

1. [https://vercel.com](https://vercel.com) → 프로젝트 대시보드 → 배포 상태 확인
2. 빌드 완료 후 프로덕션 URL에서 `/community` 접속
3. 질문 등록 → Supabase Table Editor에서 실 데이터 확인
